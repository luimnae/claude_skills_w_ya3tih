# Anti-Patterns Reference

Read this before EVERY build. These are the mistakes that make output look AI-generated,
architecturally broken, or hostile to users. Last verified: March 2026.

## 1. AI-Generated Design Slop

These patterns are dead giveaways that a design was generated by AI without human direction.
Avoid all of them.

### The Usual Suspects

**Purple gradient on white.** The single most common AI design cliché. Purple-to-blue gradient
hero section, white cards below, system font. If you catch yourself reaching for purple as
the primary color, stop and ask: does this project actually call for purple, or is it the
path of least resistance?

**Inter/Roboto/Arial everywhere.** These fonts are defaults, not choices. Using them says
"I didn't think about typography." Every project deserves a font that was chosen for its
personality, not its availability.

**Uniform padding and margin.** When every element has `p-6` or `p-8`, the design feels
mechanical. Real design has rhythm — tighter spacing in some areas, generous breathing room
in others. A card's internal padding should differ from the section padding that surrounds it.

**Cookie-cutter card grids.** Three identical cards in a row, each with an icon, heading,
paragraph, and button. Same padding, same border radius, same everything. This is a layout,
not a design. Break the grid. Make one card larger. Use asymmetry. Vary the content density.

**Overuse of rounded corners.** `rounded-2xl` on everything creates a toy-like aesthetic
unless that's intentional. Mix rounding levels — sharp containers with rounded internal
elements, or vice versa.

**Generic hero sections.** Big heading, small subtitle, two buttons (primary + ghost), maybe
a gradient blob in the background. This is not a hero — it's a placeholder. A real hero has
a point of view: a bold image, an unexpected layout, a provocative statement, a unique
visual treatment.

**Rainbow of colors with no hierarchy.** Using 5+ saturated colors with equal weight creates
visual noise. A strong palette has 1-2 dominant colors, 1 accent, and the rest are neutrals.

### How to Escape AI Slop

1. **Commit to ONE bold aesthetic choice** — If everything is "nice," nothing stands out.
   Pick one element to be extraordinary (typography, layout, motion, color, texture) and
   let it carry the design.

2. **Study real sites** — Before designing, look at 3-5 sites in the same space on
   Awwwards, Godly, or Minimal Gallery. Note what makes them distinctive. Don't copy —
   internalize their decision-making.

3. **Design with constraints** — "Make it look good" produces generic work. "Make it look
   like a 1970s Swiss magazine redesigned for Gen Z" produces memorable work. Constraints
   breed creativity.

4. **Typography first** — Choose fonts before colors. If the typography is strong, you can
   get away with a very simple color palette. The reverse is not true.

5. **Whitespace is a feature** — The most common mistake is filling every pixel. Premium
   design uses space as aggressively as it uses content. Let elements breathe.

## 2. Architecture Anti-Patterns

### "use client" on Everything

Marking entire pages or layouts as Client Components defeats the purpose of Server Components.
Every `"use client"` boundary increases the JS bundle sent to the browser.

**Fix:** Extract interactive parts into small Client Components. Keep data fetching and static
rendering in Server Components. A page with 20 components should have maybe 3-5 Client Components.

### Prop Drilling Through 10 Layers

Passing props through multiple intermediate components that don't use them.

**Fix:** Use Zustand for shared client state. Use React context for narrow trees (theme, auth).
Use Server Components for data — fetch where you render, don't prop-drill fetched data.

### One Giant Client Component for Animations

Wrapping an entire page in a single Client Component because one section needs GSAP.

**Fix:** Create small, focused animation wrapper components. `<ScrollReveal>`, `<ParallaxLayer>`,
`<MagneticButton>` — each is its own Client Component. The page layout stays as a Server Component.

### Importing Full Libraries

```js
// BAD — imports all of GSAP including unused plugins
import * as gsap from "gsap";

// GOOD — import only what you use
import gsap from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";
```

### No Error Boundaries

Animations fail silently or crash the entire app. R3F scenes fail. Network requests timeout.

**Fix:** Wrap risky components in error boundaries. Especially important for:
- R3F `<Canvas>` (WebGL can fail on old devices)
- Dynamic imports (network failures)
- Data-dependent visualizations

### Mixing Animation Systems Chaotically

Using GSAP for some hover effects, Motion for others, CSS transitions for a third set,
with no clear rationale for which system handles what.

**Fix:** Establish clear ownership:
- CSS → Simple state transitions (hover colors, focus outlines)
- Motion → Component lifecycle animations (enter, exit, layout, gestures)
- GSAP → Scroll-driven animations (everything ScrollTrigger-related)

Don't use GSAP for a button hover if Motion handles all other component animations.

### tailwind.config.js in a v4 Project

Tailwind v4 uses CSS-first configuration via `@theme`. If you're generating a v4 project
with `tailwind.config.js`, you're mixing paradigms.

**Fix:** All customization in CSS via `@theme` directive. The config file is no longer needed.

## 3. UX Anti-Patterns

### Scroll Hijacking Without Escape

Taking over the scroll completely (disabling native scroll, replacing with custom behavior)
with no way for the user to skip to content.

**Fix:** Lenis preserves native scroll behavior — it smooths, doesn't hijack. For pinned
sections, always provide a clear visual cue that content continues below. Consider a
"skip to content" link or a subtle scroll indicator.

### Animations That Block Interaction

Loading animations that prevent the user from clicking anything. Entrance animations on
critical UI (like navigation) that haven't completed.

**Fix:** Navigation should be interactive immediately. Content can animate in, but interactive
elements (buttons, links, forms) must be usable even if animations haven't finished.
Use `pointer-events: auto` on interactive elements regardless of animation state.

### Missing Reduced Motion Fallbacks

Not implementing `prefers-reduced-motion`. This isn't optional — it's an accessibility
requirement and affects users with vestibular disorders, motion sensitivity, and epilepsy.

**Fix:** Check `prefers-reduced-motion: reduce`. When active:
- Disable parallax, scrub, and pin animations
- Disable smooth scroll (Lenis)
- Show all content in final state immediately
- Keep simple fades if desired (< 300ms, no movement)

### Auto-Playing Video/Audio Without Consent

Especially on mobile where data is expensive.

**Fix:** Always require user interaction to play media. Use `poster` images for video.
If auto-playing is essential (hero background video), mute it and provide a clear
pause button.

### Infinite Scroll Without Position Memory

User scrolls through 50 items, clicks one, presses back — they're at the top again.

**Fix:** Save scroll position before navigation. Restore on return. Use `scrollRestoration`
or manual position tracking.

### Form UX Failures

- Clearing form on validation error
- Showing errors only after submit (not inline)
- No loading state on submit button
- No success confirmation

**Fix:** Inline validation on blur. Preserve input on error. Disable button + show spinner
during submit. Clear success message after 3s.

## 4. Animation Anti-Patterns

### Everything Bounces

Using `elastic` or `bounce` easing on every animation. This makes the interface feel like
a toy and quickly becomes annoying.

**Fix:** Default easing should be `power2.out` or `cubic-bezier(0.16, 1, 0.3, 1)` for most
UI animations. Reserve elastic/bounce for one or two intentional moments (a success
animation, a playful button interaction).

### Entrance Animation on Every Element

Every card, every heading, every paragraph has a fade-up-stagger animation. By the time the
user scrolls through the page, they've seen the same animation 40 times.

**Fix:** Animate sections, not individual elements. A section can have one entrance animation
that reveals its content. Or use scroll-driven animations only on the 3-4 most important
moments. The rest can appear normally.

### Duration Too Long

Animation durations over 600ms for UI interactions feel sluggish. The user is waiting for
your animation to finish before they can process the content.

**Fix:** Micro-interactions: 100-200ms. Section transitions: 300-500ms. Page transitions:
200-400ms. Scroll-scrubbed animations have no fixed duration (they follow the scroll).
Only use 600ms+ for dramatic, intentional moments (hero entrance, major reveal).

### Competing Animations

Multiple animations fighting for attention simultaneously. A parallax background, a
floating element, a typing effect, and a color transition all happening at once.

**Fix:** Choreograph. Only one "starring" animation per viewport at a time. Supporting
elements can have subtle motion, but one thing should clearly be the focus.

### No Cleanup

GSAP ScrollTriggers and timelines not killed on component unmount. Lenis not destroyed.
Motion values not cleaned up. This causes memory leaks and ghost animations.

**Fix:** Always use `useGSAP()` with `scope` (auto-cleanup). Destroy Lenis in cleanup.
Motion handles cleanup automatically if used within React lifecycle.

## 5. The "Looks Professional" Trap

There's a specific failure mode where everything is technically correct — clean code, proper
accessibility, good performance — but the design is completely forgettable. It looks like
every other well-made website. It's "professional" but not "exceptional."

**Signs you're in this trap:**
- You could swap the logo and it could be any company's site
- There's nothing a user would screenshot or share
- The design follows best practices but breaks no expectations
- You used a template structure (hero → features → testimonials → CTA → footer)
- Every section has the same visual weight

**How to escape:**
- Identify the ONE thing about this brand/product that's different. Design around that.
- Break ONE convention intentionally. Overlapping elements. Unexpected scroll behavior.
  A section that's visually "wrong" in an interesting way.
- Remove something. The design probably has too many sections. Cut the weakest one.
  Constraint creates tension, and tension creates interest.
- Add ONE moment of delight. A hover effect that makes someone smile. A scroll animation
  that reveals something unexpected. A typographic treatment that makes people pause.
- Ask: "Would I remember this site tomorrow?" If the answer is no, you're in the trap.
